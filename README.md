# Omnichannel-Retail-Sales-and-Inventory-Analytics-Dashboard
This project builds an end-to-end analytics pipeline to unify fragmented sales data from physical and digital channels. It provides a centralized dashboard to track revenue trends, optimize inventory turnover, and identify peak operational hours.

**Project Overview:**
This project focuses on bridging the data gap for multi-location retailers transitioning from traditional brick-and-mortar models to digital storefronts. By unifying fragmented data from physical Point of Sale (POS) systems and online platforms, this end-to-end data analytics pipeline provides a centralized view of business performance.

**Key Objectives:**
- Data Unification: Integrate raw sales transactions (order ID, product, quantity, and price) from both digital and physical channels into a structured format.
- Trend Analysis: Identify seasonal revenue patterns, Year-over-Year (YoY) growth, and Month-over-Month (MoM) growth across a 12-month period.
- Operational Optimization: Pinpoint peak purchasing hours and days to optimize staffing and marketing efforts.
- Inventory Intelligence: Provide actionable insights for inventory managers to increase turnover, forecast Q4 holiday spikes, and minimize dead stock.

**Technology Stack:**
- Data Cleaning: Microsoft Excel and Python (Pandas) for handling missing values and standardizing transaction logs.
- Database & Querying: SQL (MySQL/PostgreSQL) for efficient aggregation of large-scale datasets.
- Business Intelligence: Power BI or Tableau for creating interactive dashboards with drill-down capabilities by city, product category, and timeframe.

**Project Deliverables:**
- A cleaned and transformed relational dataset.
- An interactive BI dashboard featuring sales trends, geographic distribution, and time-series analysis.
- A strategic report outlining data-driven recommendations for targeted Q4 promotions and inventory procurement.
